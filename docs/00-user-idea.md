# User Idea

## What are you building?

<!-- Describe your application in a few sentences. What does it do? -->

My application is a website which uses current price data from websites such as facebook marketplace, cex and ebay alongside retailers such as scan, ebuyer and amazon to identify where the 'value' is in the used GPU market. For example if the latest Radeon GPU is the 9060 16GB and it's available new for £365 it may make better sense from a value position to buy a previous gen 7800XT if the price was less than £400. The distinction is that value changes based on pricing. This website tracks the prices and understands the relative performance between generations establishing when there's a good deal to be had.

---

## Who is it for?

<!-- Who are your target users? What's their context? -->

It's aimed at PC nerds who self-build their machines and understand that it's not always about buying the latest card and that 10fps difference isn't material if the price is signficantly different

---

## The problem it solves

<!-- What pain point or need does this address? Why does it matter? -->

There's a lot of hype in the pc market and lot's of influences and youtubers are continually pushing the latest and greatest cards. In reality those cards are not necessarily massively different for must users. This cuts through the hype and allows bargains to be found easily.

## Core features

<!-- List the key things users should be able to do. Aim for 3–7 features for an MVP.
     Don't worry about being exhaustive — the product agent will flesh these out.  -->

1. Live price - what is the current average retail price for the latest gen cards - scraped from a core set of UK retailers
2. Used price finder - scrapes used sites to find the average prices of used cards to establish a target price for a particular used version
3. Visual display - This should be a single page using some kind of graph to represent where the retail prices and the used prices overlap and show value (perahps a heat map but open to suggestions)
4. It should be two different modes Radeon vs Nvida never compare cars across manufacturers. There should also be canonical links to the OEM which lists the functionality of the cards, don't repeat taht
