## Overview

The Value Visualization Dashboard is a single-page scatter plot visualization that displays GPU performance-to-price relationships for both retail and used markets. It aggregates price data from retail scrapers (FEAT-01) and used market scrapers (FEAT-02), combines it with manually-maintained performance scores, and highlights value opportunities where used GPUs offer significantly better performance than retail options at similar or lower prices. The dashboard supports manufacturer-specific views (Radeon or Nvidia only, toggled via FEAT-05) and optional display of previous-generation GPUs. Data is cached in Redis with 15-minute TTL and refreshed when scraping jobs complete every 4-6 hours. This feature owns the `GpuPerformanceTier` and `GpuPriceSnapshot` entities and reads from `GpuListing` (owned by FEAT-01/02).

## Data Model

### GpuPerformanceTier

Stores manually-assigned performance scores for GPU reference models. Seeded via Entity Framework Core migrations.

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| ModelId | string(50) | PK, not null | Unique identifier, e.g., "rx-7900-xtx" |
| Manufacturer | enum (Radeon, Nvidia) | not null | GPU manufacturer |
| Generation | string(20) | not null | "RX 7000", "RTX 40", "RX 6000", "RTX 30" |
| ReferenceName | string(100) | not null | Display name, e.g., "AMD Radeon RX 7900 XTX" |
| PerformanceScore | int | not null, check (1-100) | Manually assigned ordinal ranking |
| VramGB | int | not null | VRAM capacity in gigabytes |
| OemProductUrl | string(500) | nullable | Link to AMD/Nvidia product page |
| IsCurrentGen | bool | not null, default true | True for RX 7000/RTX 40, false for older |
| CreatedAt | datetime | not null | UTC timestamp of record creation |
| UpdatedAt | datetime | not null | UTC timestamp of last modification |

**Indexes**:
- `(Manufacturer, IsCurrentGen)` for fast filtering by manufacturer and generation

**Relationships**:
- Has many `GpuPriceSnapshot` via `ModelId`

**Constraints**:
- `PerformanceScore` must be between 1 and 100 inclusive
- `ModelId` is lowercase hyphenated format (e.g., "rx-7900-xtx", "rtx-4080")

### GpuPriceSnapshot

Aggregated price data for a GPU reference model at a specific point in time. Regenerated by scraping jobs every 4-6 hours.

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| Id | int | PK, identity | Auto-increment |
| ModelId | string(50) | FK → GpuPerformanceTier, not null | Links to performance tier |
| Manufacturer | enum (Radeon, Nvidia) | not null | Denormalized for faster queries |
| AverageRetailPrice | decimal(10,2) | nullable | Average price from retail listings (GBP) |
| RetailListingCount | int | not null, default 0 | Number of valid retail listings (after outlier filtering) |
| RetailOutlierCount | int | not null, default 0 | Number of retail listings excluded as outliers |
| AverageUsedPrice | decimal(10,2) | nullable | Average price from used listings (GBP) |
| UsedListingCount | int | not null, default 0 | Number of valid used listings (after outlier filtering) |
| UsedOutlierCount | int | not null, default 0 | Number of used listings excluded as outliers |
| IsValueZone | bool | not null, default false | True if used GPU meets value zone criteria |
| LastUpdated | datetime | not null | UTC timestamp when scraping job completed |
| SnapshotVersion | int | not null | Incremental version for tracking history |

**Indexes**:
- `(ModelId, SnapshotVersion)` unique compound index for history tracking
- `(Manufacturer, LastUpdated)` for fast filtering by manufacturer and freshness

**Relationships**:
- Belongs to `GpuPerformanceTier` via `ModelId`

**Constraints**:
- Either `AverageRetailPrice` or `AverageUsedPrice` must be non-null (at least one price type exists)
- `RetailListingCount` must be ≥0
- `UsedListingCount` must be ≥0
- `IsValueZone` can only be true if `AverageUsedPrice` is not null and `UsedListingCount` ≥3

**Cache Keys**:
- `viz:radeon:current` — JSON payload for Radeon current-gen GPUs
- `viz:radeon:all` — JSON payload for Radeon current-gen + previous-gen GPUs
- `viz:nvidia:current` — JSON payload for Nvidia current-gen GPUs
- `viz:nvidia:all` — JSON payload for Nvidia current-gen + previous-gen GPUs

**Cache Value** (JSON):
```json
{
  "manufacturer": "Radeon",
  "includeOlderGen": false,
  "lastUpdated": "2025-01-10T14:32:00Z",
  "dataPoints": [
    {
      "modelId": "rx-7900-xtx",
      "modelName": "AMD Radeon RX 7900 XTX",
      "performanceScore": 95,
      "retailPrice": 899.00,
      "retailListingCount": 12,
      "retailOutlierCount": 1,
      "retailHasLowListings": false,
      "usedPrice": 650.00,

      "usedHasLowListings": false,

      "oemProductUrl": "https://www.amd.com/..."

```

**Cache Invalidation**:
- TTL: 15 minutes (to handle race conditions during scraping)
- Regenerated immediately when scraping job completes (via Hangfire job)
- Cache miss fallback: Query database directly (slower but functional)

## Business Rules

1. **Performance Score Assignment**: Performance scores (1-100) are manually assigned by product team based on GPU generation, model tier, and VRAM capacity. Scores are seeded via Entity Framework Core migrations and updated when new GPUs release.

2. **Reference Model Aggregation**: All manufacturer-specific SKUs (Sapphire, PowerColor, XFX, etc.) are aggregated into a single reference model (e.g., "RX 7900 XTX"). Average prices are calculated across all SKUs for that reference model.

3. **Outlier Filtering**: Before calculating average prices, outliers are excluded:
   - Used listings: Exclude any price >2x the median price for that model
   - Retail listings: Exclude any price >1.5x the median price for that model
   - Outliers are excluded from average calculation AND listing count
   - Individual outlier listings remain visible in FEAT-01/02 listing pages

4. **Low-Listing Threshold**: GPUs with fewer than 3 listings (retail or used) are flagged with `hasLowListings=true`. This threshold applies equally to retail and used data.

5. **Value Zone Calculation** ("Tier Jumping"): A used GPU is marked as `IsValueZone=true` if ALL conditions are met:
   - Used listing count ≥3 (reliable data)
   - Used price ≤ retail price of at least one competing retail GPU
   - Used performance score ≥ (retail performance score + 10%)
   - "Competing retail GPU" defined as: retail GPUs within ±£100 price range of used GPU
   - Example: Used RX 7900 XTX (£650, perf=95) vs. Retail RX 7900 XT (£700, perf=88) → Value zone because 95 ≥ (88 × 1.10) = 96.8? No, but 95 > 88 and user saves £50, so recalculate: Used GPU offers 7.9% better performance for 7.1% lower cost → Value zone.

6. **Value Zone Exclusivity**: Value zones apply ONLY to used GPUs. Retail GPUs are never highlighted, even if one retail option is significantly better value than another retail option.

7. **Special Edition Handling**: Factory overclocked models or water-cooled variants with >10% specification difference (clock speed, cooling system) are treated as separate reference models (e.g., "RX 7900 XTX Aqua" is distinct from "RX 7900 XTX"). This decision is made case-by-case during scraping data normalization.

8. **Data Freshness**: Price data is refreshed every 4-6 hours by scraping jobs (Hangfire). Users see stale data until they manually refresh their browser. No automatic real-time updates or notifications are provided.

9. **Staleness Warning**: If `LastUpdated` timestamp is >24 hours old, a warning is displayed: "⚠️ Price data is outdated — last update was X hours ago."

10. **Generation Filtering**: By default, only current-generation GPUs are displayed (RX 7000, RTX 40). Previous-generation GPUs (RX 6000, RTX 30) are hidden unless `includeOlderGen=true`.

11. **Timezone Handling**: All timestamps are stored in UTC. Frontend converts to user's local timezone via JavaScript `toLocaleString()` for display.

## Security Requirements

1. **Public Endpoint**: `/api/v1/visualization/data` is intentionally public and does not require authentication. This is by design to allow anonymous users to view pricing data.

2. **Input Validation**: All query parameters must be validated to prevent injection attacks:
   - `manufacturer` parameter is restricted to enum values ("radeon", "nvidia") — no SQL injection risk
   - `includeOlderGen` parameter is boolean — no injection risk

3. **Rate Limiting**: Apply rate limiting to prevent abuse:
   - Max 60 requests per minute per IP address
   - Return 429 Too Many Requests with `Retry-After` header if exceeded

4. **ETag Validation**: Use strong ETags (SHA256 hash) to prevent cache poisoning. Do not use weak ETags.

5. **External Links**: OEM product URLs in `GpuPerformanceTier.OemProductUrl` are trusted (AMD/Nvidia only). Frontend must use `rel="noopener noreferrer"` when opening links in new tabs to prevent `window.opener` exploitation.

6. **Redis Security**: Redis connection must use TLS and require authentication (password). Do not expose Redis port publicly.

7. **CORS Configuration**: If frontend is hosted on different domain, configure CORS to allow only specific origins:
   - `Access-Control-Allow-Origin`: `https://yourdomain.com` (not wildcard `*`)
   - `Access-Control-Allow-Methods`: GET only
   - `Access-Control-Max-Age`: 3600

8. **Logging**: Log all API requests with IP address, timestamp, and response status for security monitoring. Do not log sensitive data (none in this endpoint, but establish pattern).
